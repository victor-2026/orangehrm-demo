### 6. Edge Cases & Negative Tests

- **Assumption:** On Assign form (`/web/index.php/claim/assignClaim`)
- **Steps:**
  1. Leave Employee Name empty
  2. Select Event, Currency
  3. Click Create
- **Expected:** Validation error shown under Employee Name (e.g. "Required"). No navigation occurs. No claim created.
- **Gates:** Scope, Data

#### 6.2 Submit with empty required fields — Event
- **Assumption:** On Assign form or Submit Claim form
- **Steps:**
  1. Fill Employee Name (if on Assign form)
  2. Leave Event as "-- Select --"
  3. Fill Currency and Remarks
  4. Click Create
- **Expected:** Validation error shown under Event. No claim created.
- **Gates:** Scope, Data

#### 6.3 Submit with empty required fields — Currency
- **Assumption:** On Assign form or Submit Claim form
- **Steps:**
  1. Fill Employee and Event
  2. Leave Currency as "-- Select --"
  3. Click Create
- **Expected:** Validation error shown under Currency. No claim created.
- **Gates:** Scope, Data

#### 6.4 Submit with all fields empty
- **Assumption:** On Assign form
- **Steps:**
  1. Click Create without filling any fields
- **Expected:** Validation errors shown on all required fields (Employee, Event, Currency). Form stays. No claim created.
- **Gates:** Scope, Data

#### 6.5 Very long Remarks
- **Assumption:** On Assign form, all required fields filled
- **Steps:**
  1. Enter a Remarks string of 500+ characters
  2. Click Create

- **Gates:** Data, Evidence

#### 6.6 Invalid date format in search
- **Assumption:** On Employee Claims page

  1. Enter "13-13-2026" in From Date field

- **Expected:** Validation error on date field, or system gracefully handles as no results
- **Gates:** Scope, Data

#### 6.7 Autocomplete with no matching employee

- **Steps:**
  1. Type "ZZZNonExistentName" in Employee Name
  2. Wait for autocomplete

- **Gates:** Scope, Data

#### 6.8 Double-click Create
- **Assumption:** On Assign form, all fields filled

## Gates Checklist (3-Layer Model)

| Gate | How verified |
|------|-------------|
| Scope | Each scenario maps to a named risk (page load, table contract, create, search, validation) |
| Data | Explicit test data specified (employee names, event, currency, reference IDs) |
| State | Each scenario assumes fresh page load (no carry-over state) |
| Run | Tests executable via `npx playwright test` |
| Evidence | Screenshots on failure, toast messages asserted, table content verified |
| Review | Each failure mode is explainable (e.g. validation error = field left empty) |
